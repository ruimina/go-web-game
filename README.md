# 棋盘对战中心（围棋 + 中国象棋）

基于 `Node.js + Express + WebSocket + Canvas` 的双游戏网页对战项目。

## 功能概览

- 游戏选项卡：`围棋` / `中国象棋`
- 支持模式：
  - 本地自我对弈
  - 网页房间联机对战（实时同步）
- 地图系统：
  - 地图编辑
  - 保存/读取地图（`localStorage`）

## 围棋功能

- 标准中国围棋核心规则：
  - 落子、提子、禁入点（自杀禁着）
  - 基础打劫限制
  - 双方连续停一手后终局计分
  - 认输
- 扩展机制：
  - 不可达点（水潭）
  - 随机生成连通不可达区域（可配置区域数量和大小）
  - 自定义围棋地图（棋盘大小 + 水潭点位）

## 中国象棋功能

- 标准中国象棋走子与约束：
  - 将/士/象/马/车/炮/兵走法
  - 将帅照面约束
  - 合法性校验（不能走成己方被将）
  - 认输、无合法步判负
- 扩展机制：
  - 开局可布置中立“符”棋子
  - “符”可被吃，吃子方棋子升级
  - 升级棋子带钻石质感外观区分
- 升级能力：
  - 兵：可斜向前 1 格并吃子，且可横走
  - 炮：吃子可隔 1 子或隔 2 子
  - 车：可按后走法（直线 + 斜线）
  - 马：不受蹩马腿限制
  - 象：可按国际象棋象走法
  - 士：可按国际象棋后走法
  - 将：可按国际象棋王走法（一步八向）
- 自定义象棋地图：
  - 编辑“符”位置
  - 保存/读取地图

## 技术栈

- Node.js (ESM)
- Express
- ws (WebSocket)
- 原生 HTML/CSS/JavaScript（Canvas 渲染）

## 本地启动

### 1) 环境要求

- Node.js 18+（建议 LTS）

### 2) 安装依赖

```bash
npm install
```

### 3) 启动服务

```bash
npm start
```

默认端口：`5173`

- 页面：[http://localhost:5173](http://localhost:5173)
- 健康检查：[http://localhost:5173/health](http://localhost:5173/health)

### 4) 自定义端口（可选）

```bash
PORT=8080 npm start
```

## 基本操作

- 围棋：
  - 点击交叉点落子
  - 可使用“停一手”“计算形势”
- 中国象棋：
  - 先点击己方棋子选中，再点击目标点走子/吃子
  - 不支持“停一手”“围棋计分”按钮

## 局域网访问

同一局域网下可通过：

- `http://你的局域网IP:5173`

访问（WebSocket 同域 `/ws`）。

## 云服务器部署（概览）

1. 上传代码到云主机
2. `npm ci`
3. `npm start`（建议配合 `pm2`/`systemd`）
4. 放行端口（推荐 `80/443` + Nginx 反代到 Node 服务）

详细步骤见：[DEPLOY_ECS_NGINX.md](./DEPLOY_ECS_NGINX.md)

## 项目结构

```text
.
├── public/
│   ├── index.html          # 页面结构与双游戏控制区
│   ├── styles.css          # 样式与棋盘视觉
│   └── main.js             # 前端状态、渲染、交互、联机客户端
├── shared/
│   ├── go-rules.js         # 围棋规则与水潭生成
│   └── xiangqi-rules.js    # 象棋规则、符与升级规则
├── server.js               # Express + WebSocket 多游戏服务端
├── package.json
└── package-lock.json
```

## 备注

- 地图存储在浏览器本地 `localStorage`，不同浏览器或设备不会自动同步。
- 围棋当前为基础打劫限制，未实现超级打劫（全局同形禁着）。
